# MobiLoop MCP v0.1.0-alpha.1

Initial alpha release for guarded mobile build-test-verify automation through MCP.

## Highlights

- Guarded MCP tools for code, environment, build, Android device, iOS simulator, Appium, verification, flow memory, loop reporting, CI, and Android orchestration.
- Server-side approval enforcement for high-impact tools through `MOBILOOP_REQUIRE_APPROVAL=true`.
- Default redaction for text artifacts, command output, and MCP/CLI text responses.
- Runtime `mobiloop.config.json` validation with JSON Schema.
- CLI fallback with `mobiloop call`, `mobiloop list-tools`, and `mobiloop generate-scenarios`.
- Mock Appium integration coverage for the Appium client, flow DSL, and Android orchestrator path.
- Self-hosted Android fixture E2E workflow with optional AVD boot, device readiness checks, nightly schedule, and PR label gate.
- GHCR image publishing with SBOM artifact generation and non-blocking Trivy scan.

## Known Limitations

- The default Android orchestrator is a build-test-verify-classify loop, not an autonomous patch writer.
- Real Android fixture proof requires a configured self-hosted runner with Android SDK, AVD or physical device, Flutter, Appium 2, and UiAutomator2.
- iOS has lower-level simulator tools but no `orchestrator.run_ios_validation_loop` yet.
- Screenshot/OCR redaction and project-specific secret detectors should be handled by host policy.
- NPM publishing is not enabled in this alpha.
