# MobiLoop MCP v0.1.0-alpha.2

Follow-up alpha hardening release after real local Android validation on Genymotion.

## Highlights

- Fixed native Appium locator strategy mapping for Android UiAutomator and iOS predicate locators.
- Added a regression test for native mobile locator translation.
- Improved text redaction precision so hostnames, Android package names, class names, localhost URLs, and logcat timestamps remain readable.
- Added sanitized real-run Flutter Android evidence under `examples/artifacts/real-runs/flutter-login-android`.
- Fixed the Flutter fixture widget test import and verified it with `flutter test`.
- Tightened npm and Docker package contents to exclude generated Flutter build/cache files.

## Known Limitations

- The default Android orchestrator is a build-test-verify-classify loop, not an autonomous patch writer.
- Real Android fixture proof still requires a configured Android runner or local Android environment with Appium and UiAutomator2.
- iOS has lower-level simulator tools but no `orchestrator.run_ios_validation_loop` yet.
- Screenshot/OCR redaction and project-specific secret detectors should be handled by host policy.
- NPM publishing is not enabled in this alpha.
