# MobiLoop MCP v0.1.0-alpha.3

Alpha fixture and artifact layout release.

## Highlights

- Added a native Android Kotlin login fixture with stable resource ids and accessibility labels.
- Added `runId` / `MOBILOOP_RUN_ID` support for run-scoped evidence under `.mobiloop/runs/<runId>`.
- Updated artifact manifests, loop reports, build log collection, env preflight, and flow memory to use the same run-scoped artifact root.
- Added regression coverage for run-scoped artifact paths.
- Improved source-flow detection for native Android projects whose root is already the Gradle project.
- Sanitized GitHub summaries and PR comments when text redaction is enabled.
- Kept GHCR SBOMs as workflow artifacts instead of automatic release assets to avoid extra release-write permission.

## Container Images

Expected tags:

## Known Limits

- The native Android fixture is packaged as source; the self-hosted E2E workflow still targets the Flutter fixture.
- iOS orchestration parity remains planned.
- NPM publishing is not enabled for this alpha.
