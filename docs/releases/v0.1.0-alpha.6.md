# MobiLoop MCP v0.1.0-alpha.6

Alpha fixture coverage and runtime image update release.

## Highlights

- Added a source-only React Native login fixture.
- Included React Native fixture files in npm/GHCR package contents.
- Improved React Native source-flow analysis for screen, route, `accessibilityLabel`, placeholder, and visible-text detection.
- Merged the Docker runtime update to `node:26-bookworm-slim`.
- Kept the GitHub Actions major-version Dependabot PR open pending self-hosted Android runner validation.

## React Native Fixture

The fixture is available under:

```text
examples/mobile-fixtures/react-native-login-demo/
```

It covers:

- `testID` locators such as `login.email`, `login.password`, and `login.submit`.
- `accessibilityLabel` values for Appium-friendly discovery.
- Placeholder and visible text extraction.
- Login and home screen source-flow signals.

## Docker Runtime

The container base image now uses:

```text
node:26-bookworm-slim
```

Before merge, this was smoke-tested with:

- Docker image build.
- Dockerfile `npm test` stage.
- Runtime `node --version`.
- Basic MobiLoop CLI tool metadata calls.
