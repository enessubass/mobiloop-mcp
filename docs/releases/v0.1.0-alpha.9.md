# MobiLoop MCP v0.1.0-alpha.9

This alpha hardens iOS simulator validation for real Flutter projects on Apple Silicon hosts.

## Added

- `ios.build_app` accepts `buildSettings` and `xcodebuildArgs`.
- `orchestrator.run_ios_validation_loop` forwards those build settings into `ios.build_app`.
- Flutter iOS examples now show explicit Apple Silicon simulator build settings:

```json
{
  "buildSettings": {
    "ARCHS": "arm64",
    "EXCLUDED_ARCHS": ""
  }
}
```

## Changed

- iOS setup and build failures now return a structured orchestrator result with `passed: false`, `setupFailure: true`, `status`, `likelyRootCause`, and `nextSuggestedAction`.
- Failed iOS builds write the captured xcodebuild log to `.mobiloop` before returning.
- Long command output capture now preserves the tail of stdout/stderr so final Xcode/Gradle failure lines are retained.

## Verified

- Local format, lint, typecheck, unit, integration, audit, and package dry-run checks.
- YonetimApp Android smoke validation passed on Genymotion/Appium.
- YonetimApp iOS simulator setup failure is now captured as structured MCP evidence instead of an uncaught install/build error.

## Known Limitations

- YonetimApp iOS currently has a project-level simulator blocker: the default Flutter simulator build produced an x86_64-only app that cannot install on the current arm64 simulator, and forcing arm64 exposes native dependency/binary-slice build failures. MobiLoop now reports this cleanly, but the app dependency/build configuration still needs to be fixed before a green iOS Appium run is possible.
